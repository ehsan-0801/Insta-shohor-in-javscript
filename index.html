Javascript is the most popular language which is actually used for web. Besides this, JavaScript is a single-threaded programming language. That means JavaScript works in a single call stack method. This call stack is a data structure which works in top to down approach which means at time of execution of function it starts from the top and when it is time to return from a function it pops from the top of the function. Moreover, JavaScript is a client side scripting Language which means JavaScript runs at the client side inside the web browsers.

As we all know that JavaScript executes on single thread. It just uses some simple data structures. But sometimes it gives the illustion of multi threading. Event loop comes play a vital role here in Javascript asynchronus programming method. We know that Javascript runs in a call stack process which keeps track of all the operations. After executing the function it returns the results in a stack and event queue is responsible her for sending the new functions to the stack for processing. Here it follows the queue data structure for execution. Whenever a async function is called like setTimeout() it waits untill the specified time. Event queue runs the async function in cyclic manner and when it is time for async function to execute it excutes the function. This task is executed by event loop. It checks the stack whether it is empty or not.If it finds it empty then sends the new function to execute.
